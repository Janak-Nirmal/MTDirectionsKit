MTDAddress Class Reference

Inherits from NSObject
Declared in MTDAddress.h

Overview

MTDAddress represents the human readable address description of a waypoint. It can either be initialized and read in a normalized form (separate fields for country, county, postalCode etc.) or with a single string. Currently a single string doens’t get parsed and normalised.

descriptionWithAddressFields:

On normalised address objects this method can be used to create a description of the address only including the specified address fields (via bitmask). Valid address fields are specified in the enum MTDAddressField:

- (NSString *)descriptionWithAddressFields:(NSUInteger)addressFieldMask

Parameters

addressFieldMask

a bitmaks including the address fields to include in the description, e.g. MTDAddressFieldCountry | MTDAddressFieldCity

Return Value

a string representation of the address including the specified fields

Discussion

  • MTDAddressFieldCountry
  • MTDAddressFieldState
  • MTDAddressFieldCounty
  • MTDAddressFieldPostalCode
  • MTDAddressFieldCity
  • MTDAddressFieldStreet

On non-normalized string objects this method just returns the fullAddress string.
