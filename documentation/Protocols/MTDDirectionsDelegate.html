MTDDirectionsDelegate Protocol Reference

Conforms to NSObject
Declared in MTDDirectionsDelegate.h

Overview

The MTDDirectionsDelegate protocol defines a set of optional methods that you can implement to receive information updates about MTDirectionsKit-related stuff.

Tasks

  • – mapView:willStartLoadingDirectionsFrom:to:routeType:

    Tells the delegate that the mapView will start loading directions
  • – mapView:didFinishLoadingDirectionsOverlay:

    Tells the delegate that the specified directionsOverlay was loaded on the specified mapView. This method allows the delegate to intercept MTDirectionsKit and change the directionsOverlay by returning a different overlay than the one received as parameter
  • – mapView:didFailLoadingDirectionsOverlayWithError:

    Tells the delegate that the mapView specified wasn’t able to load the directions requested
  • – mapView:shouldActivateRoute:ofDirectionsOverlay:

    Asks the delegate whether the specified route should be activated by tapping on it. If this method isn’t implemented route activation is enabled by default.
  • – mapView:didActivateRoute:ofDirectionsOverlay:

    Tells the delegate that the user activated the specified route by tapping on it
  • – mapView:colorForRoute:ofDirectionsOverlay:

    Asks the delegate for the color for the corresponding route of the specified directionsOverlay.
  • – mapView:lineWidthFactorForDirectionsOverlay:

    Asks the delegate for the factor to multiply MKRoadWidthAtZoomScale with to compute the total overlay line width for the specified directionsOverlay. The default value, if not implemented, is 1.8f
  • – mapView:didUpdateUserLocation:distanceToActiveRoute:ofDirectionsOverlay:

    This method is an extended version of – [MKMapViewDelegate mapView:didUpdateUserLocation] and is called at exactly the same times.

mapView:didUpdateUserLocation:distanceToActiveRoute:ofDirectionsOverlay:

This method is an extended version of – [MKMapViewDelegate mapView:didUpdateUserLocation] and is called at exactly the same times.

- (void)mapView:(id<MTDMapView>)mapView didUpdateUserLocation:(CLLocation *)location distanceToActiveRoute:(CGFloat)distanceToActiveRoute ofDirectionsOverlay:(MTDDirectionsOverlay *)directionsOverlay

Parameters

mapView

the map view that is tracking the user’s location

location

the location object representing the user’s latest location

distanceToActiveRoute

the distance in points between the userLocation and the active route on the mapView

directionsOverlay

the overlay that contains the active route

Discussion

From Apple’s documentation: While the showsUserLocation property is set to YES, this method is called whenever a new location update is received by the map view. This method is also called if the map view’s user tracking mode is set to MKUserTrackingModeFollowWithHeading and the heading changes.

This method is not called if the application is currently running in the background. If you want to receive location updates while running in the background, you must use the Core Location framework.
